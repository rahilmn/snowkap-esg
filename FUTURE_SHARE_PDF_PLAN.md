# Future Enhancement: Share as PDF — Full Intelligence Report

## Status: PLANNED (not yet implemented)

## Overview

A "Share as PDF" button inside ArticleDetailSheet that generates a professional PDF report containing the full intelligence analysis across all 3 perspectives (CFO, CEO, ESG Analyst), with an explanation of how the analysis was computed (12-stage pipeline, primitives cascade, ontology coverage).

## PDF Report Structure (~4-6 pages)

### Page 1: Cover + Executive Summary
- Snowkap logo + "ESG Intelligence Report"
- Article title, source, date, company name
- Headline (from deep_insight)
- Materiality badge (CRITICAL/HIGH/MODERATE/LOW)
- Financial exposure (computed, with "from article" vs "engine estimate" flag)
- Key risk + Top opportunity

### Page 2: Three-Perspective Analysis

| Section | Content |
|---------|---------|
| **CFO VIEW** | Headline, Impact Grid (Financial/Regulatory/Strategic), What Matters (2-3 bullets), Action (1-2 bullets) |
| **CEO VIEW** | Headline, Impact Grid, What Matters (2-3 bullets), Action (1-2 bullets) |
| **ESG ANALYST VIEW** | Headline, Impact Grid, What Matters (2-3 bullets), Action (1-3 bullets) |

### Page 3: Financial Impact & Risk
- Financial Timeline (immediate / structural / long-term)
- Risk Assessment table (category, probability, exposure, score, level)
- ESG Relevance Scores (6 dimensions with score + rationale)

### Page 4: AI Recommendations
- Each recommendation: type badge, title, description, framework section, budget, ROI%, payback, deadline, urgency, responsible party
- Profitability link explanation

### Page 5: Methodology & Computation Trace
- "How this analysis was computed"
- 12-stage pipeline overview (which stages are ontology-driven, which are LLM)
- Causal primitive cascade trace: e.g., "CL->OX: +₹5.04 Cr (β=0.10, step function, lag=0-2q)"
- Company calibration used: revenue, opex, energy share, etc.
- Ontology coverage statement: "97% ontology-driven, 3% LLM"
- Framework alignment (which frameworks triggered and why)

### Footer (all pages)
"Generated by Snowkap ESG Intelligence Engine | {date} | Confidential"

## Technical Approach

### Backend: `engine/export/pdf_report.py` (NEW)
- Use `reportlab>=4.0` for programmatic PDF generation (pure Python, no external deps)
- `generate_report_pdf(payload: dict, company: Company) -> bytes`
- Reads the full insight JSON and renders all 5 pages

### API: `POST /api/news/{article_id}/export-pdf`
```python
@router.post("/news/{article_id}/export-pdf")
def export_pdf(article_id: str, _: None = Depends(require_auth)):
    row = sqlite_index.get_by_id(article_id)
    payload = _load_payload(row.get("json_path"))
    company = get_company(row.get("company_slug", ""))
    pdf_bytes = generate_report_pdf(payload, company)
    return Response(
        content=pdf_bytes,
        media_type="application/pdf",
        headers={"Content-Disposition": f"attachment; filename=snowkap-{article_id}.pdf"}
    )
```

### Frontend: Button in ArticleDetailSheet
```tsx
<button onClick={() => handleExportPdf(article.id)} title="Share as PDF">
  Share as PDF
</button>
```
- Handler fetches blob from API and triggers browser download
- Button appears in hero card action area (around line 735)
